Go-Live Checklist

The Go-Live Checklist tool helps you validate that your AEM project is ready for launch. This checklist is based on the official go-live checklist.

Enter your project details below to validate the checklist items that can be automatically checked.

Note: If running locally, some automated checks may be restricted by CORS policies. Deploy this tool to your AEM domain or use the manual check links provided in error messages.

Enter your production domain to check additional items (e.g. www.yourdomain.com)

Checklist Results

Steps Before Go-Live

Content and Design QA

⚠️ Content and Design Review Manual Check

Verify that content and design conform to specifications on your .aem.live domain.

  • Check for accessibility requirements
  • Validate SEO requirements
  • Review design specifications

Performance Validation

⏳ Lighthouse Score Pending

Every AEM project should achieve a Lighthouse score of 100 for mobile and desktop.

Analytics Validation

⚠️ Analytics Setup Manual Check

Verify that analytics and martech stack are firing correctly.

  • Check analytics dashboards for visitor data
  • Expect baseline metrics to change after launch
  • Coordinate with analysts about metric adjustments

RUM Instrumentation

⏳ Real User Monitoring (RUM) Pending

RUM should be instrumented to track performance metrics before and after launch.

Legacy Redirects

⏳ Redirects Configuration Pending

Check for redirects.xlsx in SharePoint or redirects in Google Drive.

Sitemap & Robots

⏳ Sitemap Pending

Verify that sitemap is generated and accessible.

⏳ Robots.txt Pending

Check that robots.txt allows crawlers and references sitemap.

Canonical URLs

⏳ Canonical Tags Pending

Verify canonical URLs return 2xx status and are correctly implemented.

Favicon

⏳ Favicon Pending

Verify that favicon is configured for the site.

CORS Headers

⏳ CORS Configuration Pending

Check if proper CORS headers are configured for cross-origin requests.

Authentication for Authors

⚠️ Author Authentication Manual Check

Decide if you want to control who can preview and publish documents.

See Authentication for Authors documentation.

SharePoint Access

⚠️ SharePoint Access Configuration Manual Check

If using SharePoint, configure dedicated access that you control.

See SharePoint Setup documentation.

CDN Configuration

⏳ CDN Setup Pending

Configure CDN to point to .aem.live endpoint.

Use the Push Invalidation tool to validate your CDN configuration.

Push Invalidation Setup

⏳ Push Invalidation Pending

Ensure push invalidation is properly set up for production CDN.

Notify Engineering On-Call

⚠️ Engineering Notification Manual Check

Send an email to aemgolives@adobe.com with:

  • .aem.live URL(s) going live
  • Production domain
  • Planned go-live date and time
  • Primary contact person
  • Teams or Slack channel for collaboration

Post Go-Live Validation

Performance Validation

⚠️ Production Performance Manual Check

Validate Lighthouse score of 100 on production domain after go-live.

Check for performance impacts from CDN layer (HTTP/1.1 vs HTTP/2, caching, bot detection).

Google Search Console

⚠️ Search Console Monitoring Manual Check

Monitor Google Search Console for weeks after go-live:

  • Coverage report and indexing status
  • Total clicks and impressions
  • Backlinks changes
  • Crawl errors

Martech and Analytics Validation

⚠️ Production Analytics Manual Check

Verify all martech tags fire correctly on production domain.

Ensure analytics data is being collected continuously.

404 Report

⚠️ 404 Monitoring Manual Check

Monitor 404 errors after migration and create redirects as needed.

Check site analytics and Slack bot reports for 404s.
