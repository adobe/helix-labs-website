Enhanced Snapshot Admin Palette

Features

  • Multi-Snapshot Selection: Users can now select from multiple available snapshots instead of being limited to just the 'default' snapshot
  • Dynamic Snapshot Loading: The palette automatically loads all available snapshots for the project
  • Auto-Selection: If a 'default' snapshot exists, it will be automatically selected for convenience
  • Enhanced UI: Added a dropdown selector at the top of the palette for snapshot selection
  • Contextual Status: Page and review status updates based on the selected snapshot
  • Modern Implementation: Uses modern JavaScript utilities without external framework dependencies

Usage

To use the enhanced palette:

  1. Open the palette.html file with URL parameters: ?referrer=[PAGE_URL]&owner=[OWNER]&repo=[REPO]
  2. Select a snapshot from the dropdown at the top
  3. The palette will update to show the status of the current page in relation to the selected snapshot
  4. Use the Add/Update/Remove buttons to manage the page in the selected snapshot
  5. Use the review workflow buttons if you're on a review site

Integration

The palette can be integrated into other tools or used as a popup by:

  • Opening it in an iframe or popup window
  • Passing the appropriate URL parameters
  • Styling it to match your application's theme

Example URL:

/tools/snapshot-admin/palette.html?referrer=https://main--site--org.aem.page/path&owner=org&repo=site&reviewHost=custom.reviews.com&liveHost=custom.live.com
